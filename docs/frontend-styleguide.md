# Frontend Style Guide

## 1. اصول طراحی

### 1.1 هویت بصری
- **رنگ اصلی:** آبی کاربنی عمیق `#2A2A81` (اعتماد، تازگی).
- **رنگ تاکیدی:** طلایی زعفرانی گرم `#F9A825` برای CTA و عناصر کلیدی.
- **پس‌زمینه‌های مکمل:** کرم نرم `#FFF8E1` و خاکستری ذغالی `#424242` برای کنتراست.
- **حس و حال:** دوستانه، مطمئن، با موتیف‌های الهام‌گرفته از هنر فارسی.

### 1.2 تایپوگرافی
| نقش متن | فونت | اندازه |
|---------|------|--------|
| سرفصل‌های اصلی | وزیر یا شبنم (Bold) | 26px |
| عنوان کارت / محصول | وزیر یا شبنم (Medium) | 18px |
| متن بدنه | وزیر یا شبنم (Regular) | 15px |
- فاصله‌گذاری بین حروف و خطوط مطابق استاندارد فارسی.
- پشتیبانی کامل RTL/LTR بسته به زبان انتخابی.

---

## 2. Layout و ترکیب‌بندی

### 2.1 ساختار کلی
- **چیدمان کارت‌محور** با Flow راست‌به‌چپ (RTL-first).
- **Grid System:** پایه 12 ستونه با گاتر 16px در دسکتاپ، 12px در موبایل.
- **Viewports:** Mobile-first؛ نقاط شکست 576px، 768px، 1024px، 1280px.
- **Header چسبنده:** شامل لوگو سمت راست، جستجو راست‌چین، انتخابگر زبان/ارز، آیکون حساب و سبد خرید.

### 2.2 Card Components
- گوشه‌های گرد 10px.
- سایه ملایم: `0 3px 10px rgba(0,0,0,0.08)`.
- تصویر محصول سمت راست کارت، جزئیات و قیمت سمت چپ (در RTL).
- نمایش قیمت اصلی/تخفیف با رنگ‌های مشخص (`#F44336` برای قیمت تخفیفی).

---

## 3. ناوبری و تعامل

### 3.1 Navigation
- **Mega Menu RTL:** بازشدن از راست با دسته‌بندی‌های چندسطحی.
- **Breadcrumbs:** راست‌چین با جداکننده «›».

### 3.2 عناصر کنترلی
- **Buttons:** حداقل ارتفاع 48px، حالت‌های Primary (آبی)، Secondary (طلایی)، Tertiary (شفاف با خط آبی).
- **Iconography:** آیکون‌های الهام‌گرفته از خوشنویسی فارسی؛ استفاده از بسته سفارشی SVG.
- **Inputs:** فیلدهای دارای برچسب بالایی، آیکون انتهایی راست، حالت خطا با متن قرمز `#D32F2F`.

---

## 4. تجربه کاربری

### 4.1 سبد و پرداخت
- آیکون سبد خرید ثابت در بالا-راست با Badge تعداد آیتم.
- امکان سفارش مجدد یک‌کلیکی از تاریخچه.
- نمایش مراحل Checkout در نوار پیشرفت 4 مرحله‌ای (سبد، آدرس، پرداخت، تایید).

### 4.2 بازخورد و تعامل
- Micro-interactions با انیمیشن 0.3s ease RTL.
- Toast Notifications در بالا-چپ (برای RTL)، حداکثر 3 همزمان.
- پیام‌های خطا/موفقیت به فارسی واضح و خوانا.

---

## 5. بی‌درنگی و PWA

### 5.1 PWA
- Service Worker با استراتژی Cache-first برای Assets و Stale-while-revalidate برای داده‌ها.
- Offline Placeholder برای صفحه اصلی و جزئیات محصول.
- Install Prompt بومی (Add to Home Screen) با آیکون 512px.

### 5.2 عملکرد
- بارگذاری اولیه زیر 3 ثانیه.
- تصاویر WebP/AVIF با Lazy Loading.
- استفاده از Resource Hints (preload, preconnect) برای API Gateway.

---

## 6. دسترس‌پذیری

- تطابق با WCAG 2.1 سطح AA.
- کنتراست حداقل 4.5:1 برای متن.
- فوکوس واضح با Outline 2px طلایی.
- پشتیبانی کامل کیبورد و Screen Reader (ARIA attributes).
- رعایت استانداردهای RTL برای Screen Reader (استفاده از `dir=rtl` و `lang` صحیح).

---

## 7. المان‌های ویژه صفحه اصلی

- **Featured Products Component:** اسلایدر RTL با سه کارت در دسکتاپ، یک کارت در موبایل.
- **Campaign Banner:** پس‌زمینه گرادیان آبی→طلایی، تایمر شمارش معکوس دیجیتال.
- **Public Announcements:** کارت‌های اطلاعاتی با آیکون اعلان، پس‌زمینه کرم.

---

## 8. پیاده‌سازی فنی (ASP.NET Core MVC + Razor)

- استفاده از Layout مشترک (`_Layout.cshtml`) با پارتشیال‌های Header/Footer.
- ViewComponent‌ها برای `FeaturedProducts`, `CampaignCarousel`, `WalletSummary`.
- CSS سازماندهی‌شده بر اساس BEM با Sass (در صورت نیاز).
- مدیریت زبان با Resource فایل‌ها و Tag Helperهای محلی‌سازی.
- Service Worker و Manifest در مسیر `wwwroot/pwa/`.

---

## 9. تست و تضمین کیفیت

- تست UI با Playwright (سناریوهای RTL/LTR، چند زبان/ارز، PWA offline).
- تست Snapshot برای کامپوننت‌های Razor بحرانی.
- بررسی خودکار A11y (Lighthouse, axe-core) در pipeline.
